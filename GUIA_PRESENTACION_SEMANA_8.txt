PASOS PREVIOS A LA PRESENTACIÓN
================================================================================

1. INSTALAR LIBRERÍAS REQUERIDAS
   ------------------------------
   Abrir terminal en: C:\xampp\htdocs\gestion_citas

   Ejecutar:
   composer require tecnickcom/tcpdf
   composer require phpoffice/phpspreadsheet

   Verificar que se hayan instalado correctamente en vendor/


2. INICIAR SERVICIOS
   ------------------
   - Iniciar XAMPP
   - Activar Apache
   - Activar MySQL


3. VERIFICAR BASE DE DATOS
   -----------------------
   - Asegurarse de que la base de datos tenga citas registradas
   - Idealmente tener citas de diferentes estados
   - Tener citas en diferentes fechas para mostrar gráficas


4. PREPARAR NAVEGADOR
   -------------------
   - Abrir navegador
   - Ir a: http://localhost/gestion_citas/public/
   - Iniciar sesión como administrador

PARTE 1: DASHBOARD DE REPORTES (5 minutos)

1. Acceso:
   - Desde el dashboard admin

2. Mostrar KPIs:

   - Ingresos totales generados"

3. Demostrar Filtros:
   - Cambiar el rango de fechas
   - Click en "Filtrar"
   - Mostrar cómo se actualizan los datos

4. Explicar Gráfica:
   "Esta gráfica de evolución muestra la tendencia de citas por día,
   con tres líneas para total, completadas y canceladas. Nos permite
   identificar patrones y días con mayor demanda."

5. Mostrar Tablas de Resumen:

PREGUNTAS FRECUENTES Y RESPUESTAS
================================================================================

P: ¿Los reportes se actualizan en tiempo real?
R: Sí, cada vez que se filtra por fecha, se consulta la base de datos
   y se muestran los datos actualizados.

P: ¿Se pueden exportar todos los reportes?
R: Sí, todos los reportes tienen opciones de exportación a PDF y Excel.

P: ¿Las gráficas son interactivas?
R: Sí, todas las gráficas usan Chart.js y tienen tooltips interactivos
   al pasar el mouse.

P: ¿Funciona en móviles?
R: Sí, todo el diseño es responsive y se adapta a cualquier dispositivo.

P: ¿Otros roles pueden acceder a reportes?
R: No, solo el administrador tiene acceso al módulo de reportes completo.
   Está protegido con filtros de autenticación.

P: ¿Qué pasa si no hay datos?
R: El sistema muestra mensajes claros como "No hay citas en este período"
   y los KPIs muestran 0.

P: ¿Se pueden programar reportes automáticos?
R: No en esta versión, pero está listado como mejora futura en la
   documentación.
